Modelagem Visual de Software

Status: Cursado

Disciplina voltada à representação visual de requisitos, comportamentos e estruturas de sistemas por meio de notações como UML, BPMN e SysML.

  • → Diagrama de Requisitos (SysML): representa graficamente as necessidades e funcionalidades do sistema a partir da perspectiva dos stakeholders.
  • → Diagrama de Casos de Uso de Negócios (UML): ilustra como os processos de negócios se relacionam com os atores externos, focando em alto nível.
  • → Diagrama de Caso de Uso (UML): mostra como os usuários interagem com o sistema por meio de funcionalidades específicas.
  • → Diagrama de Classe (UML): define a estrutura do sistema, com classes, atributos, métodos e seus relacionamentos.
  • → Diagrama de Sequência (UML): mostra a ordem das interações entre objetos ao longo do tempo.
  • → Diagrama de Estado (UML): representa os diferentes estados de um objeto durante seu ciclo de vida.
  • → Diagrama de Atividades (UML): mostra o fluxo de atividades ou ações dentro de um processo ou funcionalidade.
  • → Diagrama de Fluxo (BPMN): representa graficamente processos de negócio com eventos, decisões e tarefas.
  • → Diagrama de Estado (UML) - Reforço: reforça os conceitos de transições entre estados e reações do sistema a eventos.
Paradigmas da Engenharia de Software

Status: Cursado

Disciplina focada na evolução dos modelos e abordagens de desenvolvimento de software, destacando a importância dos requisitos, qualidade, gestão de riscos e metodologias que conectam tecnologia à entrega de valor.

  • → História e Paradigmas: compreensão do contexto, rupturas e evolução da Engenharia de Software.
  • → Engenharia de Requisitos: foco em organização, governança e definição estruturada de requisitos.
  • → Qualidade de Software: padrões, ética, valor ao cliente e impacto da qualidade no ciclo de vida.
  • → Gerência de Configuração: controle de versões, rastreabilidade e integridade dos artefatos.
  • → Riscos em Projetos: análise de riscos com base na ISO 31000 e no conceito de antifragilidade.
  • → Cronograma e Estimativas: planejamento estratégico com uso de PERT e boas práticas de estimativa.
  • → Métricas de Software: decisões orientadas por dados e gestão baseada em evidências.
  • → Testes de Software: cultura TestOps, resiliência e mentalidade antifrágil no desenvolvimento.

Projeto de Estudo: Classificador de Nível de Herói

Desenvolvido como parte da Formação em Lógica de Programação da DIO, este projeto simples utiliza estruturas básicas da programação para classificar um personagem com base em sua experiência (XP).

Requisitos do Desafio

  • Variáveis
  • Operadores
  • Laços de repetição
  • Estruturas de decisão (if/else)

Objetivo

  • Menor que 1.000 → Ferro
  • 1.001 a 2.000 → Bronze
  • 2.001 a 5.000 → Prata
  • 5.001 a 7.000 → Ouro
  • 7.001 a 8.000 → Platina
  • 8.001 a 9.000 → Ascendente
  • 9.001 a 10.000 → Imortal
  • 10.001 ou mais → Radiante

Código Gerado


// Classificador de Herói
let nome = "Spartacus"
let quantidadeXp = 7634
let nivel = ""

if (quantidadeXp < 1000) {
    nivel = "Ferro"
} else if (quantidadeXp >= 1001 && quantidadeXp <= 2000) {
    nivel = "Bronze"
} else if (quantidadeXp >= 2001 && quantidadeXp <= 5000) {
    nivel = "Prata"
} else if (quantidadeXp >= 5001 && quantidadeXp <= 7000) {
    nivel = "Ouro"
} else if (quantidadeXp >= 7001 && quantidadeXp <= 8000) {
    nivel = "Platina"
} else if (quantidadeXp >= 8001 && quantidadeXp <= 9000) {
    nivel = "Ascendente"
} else if (quantidadeXp >= 9001 && quantidadeXp <= 10000) {
    nivel = "Imortal"
} else {
    nivel = "Radiante"
}

console.log("O Herói de nome " + nome + " está no nível de " + nivel)
